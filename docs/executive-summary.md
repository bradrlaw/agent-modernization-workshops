# Executive Summary – Agent Modernization Program

## Purpose

This program enables teams to transition from deprecated Azure Bot Framework SDK–based bots
to Microsoft's current agent stack: **Copilot Studio** (low/no-code), **Azure AI Foundry**
(pro-code), and the **Microsoft 365 Agents SDK** (runtime and channels).

## Program at a Glance

- 8 bi-weekly workshops paired with hands-on labs
- Parallel environment readiness plan to avoid delivery blockers
- Hybrid and multi-agent patterns included to support fusion teams

## Primary Outcomes

1. A clear **decision framework** for when to use Copilot Studio vs Foundry vs Agents SDK
2. **Working reference agents** built along the spectrum from low-code to pro-code
3. **Operational readiness**: evaluation, observability, and governance-aligned environments

## Critical Dependency Timeline

| Milestone | Action |
|---|---|
| Before Week 2 | Power Platform environment + Copilot Studio enabled |
| Before Week 3 | Azure AI Foundry + model access approvals |
| Before Week 4 | App registrations + Teams publishing permissions |

## Result

Teams exit with approved environments, deployable agents, and a repeatable modernization
playbook for continued agent delivery beyond the program.
